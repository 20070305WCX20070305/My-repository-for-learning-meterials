# 复杂度分析、栈与链表

## 一、复杂度分析

### 1.1 时间复杂度

时间复杂度衡量算法执行时间随输入规模增长的变化趋势，用大O记号表示（最坏情况上界）。

| 符号 | 名称 | 说明 |
|------|------|------|
| O(1) | 常数阶 | 执行时间与输入规模无关 |
| O(log n) | 对数阶 | 常见于二分查找 |
| O(n) | 线性阶 | 遍历所有元素 |
| O(n log n) | 线性对数阶 | 常见于高效排序 |
| O(n²) | 平方阶 | 双层嵌套循环 |
| O(2^n) | 指数阶 | 暴力递归 |
| O(n!) | 阶乘阶 | 全排列问题 |

### 1.2 空间复杂度

空间复杂度衡量算法运行所需的额外内存空间。

- **O(1)**：仅使用固定数量的额外变量
- **O(n)**：需要与输入规模成比例的额外存储

### 1.3 分析方法

- 忽略常数项和低阶项
- 循环次数决定复杂度层级
- 递归算法的时间复杂度 = 递归深度 × 每层时间

## 二、栈（Stack）

### 2.1 定义

栈是一种**后进先出（LIFO, Last In First Out）**的线性数据结构。元素只能从栈顶插入（push）和移除（pop）。

### 2.2 时间复杂度

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| push | O(1) | 在栈顶插入 |
| pop | O(1) | 移除栈顶元素 |
| peek | O(1) | 访问栈顶元素 |
| is_empty | O(1) | 判空 |
| size | O(1) | 返回元素个数 |

### 2.3 空间复杂度

- **O(n)**：n 为栈中元素个数

### 2.4 面向对象实现（Python）

```python
class Stack:
    def __init__(self):
        self._items = []

    def push(self, item):
        self._items.append(item)

    def pop(self):
        if self.is_empty():
            raise IndexError("pop from empty stack")
        return self._items.pop()

    def peek(self):
        if self.is_empty():
            raise IndexError("peek from empty stack")
        return self._items[-1]

    def is_empty(self):
        return len(self._items) == 0

    def size(self):
        return len(self._items)

    def __repr__(self):
        return f"Stack({self._items})"

    def __len__(self):
        return self.size()
```

### 2.5 应用场景

- 函数调用栈（递归）
- 表达式求值（中缀转后缀）
- 括号匹配
- 浏览器前进/后退

## 三、链表（Linked List）

### 3.1 定义

链表是一种由若干节点（Node）组成的线性数据结构，每个节点包含数据和指向下一个节点的指针。链表分为**单向链表**和**双向链表**。

### 3.2 时间复杂度

| 操作 | 单向链表 | 双向链表 | 说明 |
|------|---------|---------|------|
| 头插 | O(1) | O(1) | 在头部插入 |
| 头删 | O(1) | O(1) | 删除头节点 |
| 尾插（无尾指针） | O(n) | O(1) | 在尾部插入 |
| 尾删（无尾指针） | O(n) | O(1) | 删除尾节点 |
| 按值查找 | O(n) | O(n) | 遍历直到找到 |
| 按索引访问 | O(n) | O(n) | 必须从头遍历 |

### 3.3 空间复杂度

- **O(n)**：n 为节点个数
- 每个节点额外存储 1~2 个指针，空间开销略高于数组

### 3.4 面向对象实现（Python）

#### 单向链表

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

    def __repr__(self):
        return f"Node({self.data})"


class SinglyLinkedList:
    def __init__(self):
        self._head = None
        self._size = 0

    def is_empty(self):
        return self._head is None

    def size(self):
        return self._size

    def push_front(self, data):
        node = Node(data)
        node.next = self._head
        self._head = node
        self._size += 1

    def pop_front(self):
        if self.is_empty():
            raise IndexError("pop from empty list")
        data = self._head.data
        self._head = self._head.next
        self._size -= 1
        return data

    def push_back(self, data):
        node = Node(data)
        if self.is_empty():
            self._head = node
        else:
            cur = self._head
            while cur.next:
                cur = cur.next
            cur.next = node
        self._size += 1

    def pop_back(self):
        if self.is_empty():
            raise IndexError("pop from empty list")
        if self._head.next is None:
            data = self._head.data
            self._head = None
        else:
            cur = self._head
            while cur.next.next:
                cur = cur.next
            data = cur.next.data
            cur.next = None
        self._size -= 1
        return data

    def find(self, value):
        cur = self._head
        while cur:
            if cur.data == value:
                return True
            cur = cur.next
        return False

    def to_list(self):
        result = []
        cur = self._head
        while cur:
            result.append(cur.data)
            cur = cur.next
        return result

    def __repr__(self):
        return " -> ".join(str(d) for d in self.to_list()) + " -> None"

    def __len__(self):
        return self._size
```

#### 双向链表

```python
class DoublyNode:
    def __init__(self, data):
        self.data = data
        self.prev = None
        self.next = None

    def __repr__(self):
        return f"DoublyNode({self.data})"


class DoublyLinkedList:
    def __init__(self):
        self._head = None
        self._tail = None
        self._size = 0

    def is_empty(self):
        return self._head is None

    def size(self):
        return self._size

    def push_front(self, data):
        node = DoublyNode(data)
        if self.is_empty():
            self._head = self._tail = node
        else:
            node.next = self._head
            self._head.prev = node
            self._head = node
        self._size += 1

    def pop_front(self):
        if self.is_empty():
            raise IndexError("pop from empty list")
        data = self._head.data
        self._head = self._head.next
        if self._head:
            self._head.prev = None
        else:
            self._tail = None
        self._size -= 1
        return data

    def push_back(self, data):
        node = DoublyNode(data)
        if self.is_empty():
            self._head = self._tail = node
        else:
            node.prev = self._tail
            self._tail.next = node
            self._tail = node
        self._size += 1

    def pop_back(self):
        if self.is_empty():
            raise IndexError("pop from empty list")
        data = self._tail.data
        self._tail = self._tail.prev
        if self._tail:
            self._tail.next = None
        else:
            self._head = None
        self._size -= 1
        return data

    def to_list(self):
        result = []
        cur = self._head
        while cur:
            result.append(cur.data)
            cur = cur.next
        return result

    def __repr__(self):
        return " <-> ".join(str(d) for d in self.to_list()) + " <-> None"

    def __len__(self):
        return self._size
```

### 3.5 链表 vs 数组

| 对比项 | 数组 | 单向链表 | 双向链表 |
|-------|------|---------|---------|
| 索引访问 | O(1) | O(n) | O(n) |
| 头插/头删 | O(n) | O(1) | O(1) |
| 尾插/尾删 | O(1)* | O(n) | O(1) |
| 空间开销 | 低 | 中 | 高（额外前驱指针） |
| 内存连续性 | 连续 | 不连续 | 不连续 |

> *数组尾插可能触发扩容 O(n)，均摊后为 O(1)

## 四、栈与链表的综合对比

| 维度 | 栈（基于数组） | 栈（基于链表） |
|------|-------------|-------------|
| 查找 | 不支持 | 不支持 |
| 插入/删除 | 仅顶部 O(1) | 仅顶部 O(1) |
| 空间利用 | 连续，可能预分配浪费 | 按需分配，指针有额外开销 |
| 缓存友好 | 是 | 否 |

| 维度 | 链表 | 数组栈 |
|------|------|-------|
| 随机访问 | ❌ O(n) | ✅ O(1) |
| 动态扩容 | ✅ 天然支持 | ❌ 需重新分配 |
| 内存碎片 | 易产生 | 连续内存 |

### 4.1 如何选择

- 只需后进先出操作 → **栈**
- 需要频繁头部插入/删除 → **单向链表**
- 需要频繁两端操作 → **双向链表（或双端队列）**
- 需要快速随机访问 → **数组**
