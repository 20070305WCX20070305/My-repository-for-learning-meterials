# 哈希查找

## 基本概念

**哈希查找** 是通过关键字直接访问数据的一种查找技术，时间复杂度可达到 **O(1)**。

| 概念 | 说明 |
|------|------|
| **散列 (Hashing)** | 将关键字映射到存储位置的过程 |
| **散列表 (Hash Table)** | 按散列函数建立的对应关系存储数据的结构 |
| **散列函数 (Hash Function)** | 关键字 → 存储地址的映射函数 `addr = H(key)` |
| **冲突 (Collision)** | 多个不同关键字映射到同一位置 |
| **同义词 (Synonym)** | 产生冲突的多个关键字 |

---

## 常见散列函数

### 1. 直接定址法
```
H(key) = a * key + b
```
适用于关键字分布连续，不常用。

### 2. 除留余数法（最常用）
```
H(key) = key % p
```
**p 的选择至关重要：** 一般取**不大于表长的最大质数**或**素数**。

### 3. 数字分析法
取关键字中分布较均匀的几位作为地址。

### 4. 平方取中法
关键字平方后取中间几位。

### 5. 折叠法
将关键字分割后叠加求和。

---

## 处理冲突的方法

### 开放地址法

```
Hi = (H(key) + di) % m
```

| 探测方式 | di 序列 | 说明 |
|----------|---------|------|
| **线性探测** | 1, 2, 3, ... | 易产生**堆积**现象 |
| **二次探测** | 1², -1², 2², -2², ... | 缓解堆积，但可能探测不到所有位置 |
| **伪随机探测** | 伪随机数序列 | 实现复杂 |

**线性探测示例：**

散列函数 `H(key) = key % 7`，插入 50, 21, 58, 29, 36：

```
H(50) = 50 % 7 = 1  →  [1] 放 50
H(21) = 21 % 7 = 0  →  [0] 放 21
H(58) = 58 % 7 = 2  →  [2] 放 58
H(29) = 29 % 7 = 1  →  [1] 冲突，[2] 冲突，[3] 放 29
H(36) = 36 % 7 = 1  →  [1][2][3] 都冲突，[4] 放 36

最终：21 50 58 29 36
下标：0  1  2  3  4
```

### 链地址法（拉链法）

每个槽位是一个链表头，冲突元素挂在链表上。

```
散列表：[0] → 21
       [1] → 50 → 29 → 36
       [2] → 58
       [3] → ∅
```

**优点：** 不会堆积，删除方便，适合冲突多的情况。

---

## 查找成功 vs 查找失败

```
查找成功 ASL = 所有关键字比较次数之和 / 关键字个数
查找失败 ASL = 所有失败位置探测次数之和 / 表长
```

（ASL = Average Search Length，平均查找长度）

---

## 装填因子

```
α = 表中记录数 / 散列表长度
```

- α 越大，冲突概率越高，效率越低
- 通常控制 α 在 **0.5 ~ 0.8** 之间
- 超过阈值时需要**动态扩容**（rehash）

---

## 完整示例

```
关键字序列：{19, 14, 23, 1, 68, 20, 84, 27, 55, 11, 10, 79}
散列函数：H(key) = key % 13
处理冲突：线性探测
表长：16
```

计算每个关键字地址：
```
H(19)=6  H(14)=1  H(23)=10  H(1)=1→2  H(68)=3  H(20)=7
H(84)=6→7→8  H(27)=1→2→3→4  H(55)=3→4→5  H(11)=11
H(10)=10→11→12  H(79)=1→2→3→4→5→6→7→8→9
```

---

## 优缺点总结

| 优点 | 缺点 |
|------|------|
| 查找/插入/删除平均 O(1) | 散列函数设计依赖数据分布 |
| 实现简单 | 需要处理冲突，最坏 O(n) |
| 对大表效率优势明显 | 不支持范围查找（如 BETWEEN） |
| | 需要扩容 rehash 开销 |
