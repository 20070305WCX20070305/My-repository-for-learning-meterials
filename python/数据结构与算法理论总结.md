## 一、复杂度分析

### 1.1 大 O 表示法

描述算法运行时间随输入规模增长的**上界**，忽略常数因子和低阶项。

| 记号 | 名称 | 示例 |
|------|------|------|
| O(1) | 常数阶 | 数组随机访问 |
| O(log n) | 对数阶 | 二分查找 |
| O(n) | 线性阶 | 数组遍历 |
| O(n log n) | 线性对数阶 | 归并排序、堆排序 |
| O(n²) | 平方阶 | 冒泡排序、嵌套循环 |
| O(2ⁿ) | 指数阶 | 递归斐波那契（朴素） |
| O(n!) | 阶乘阶 | 旅行商问题（暴力） |

### 1.2 常见复杂度大小关系

```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)
```

### 1.3 渐进符号

- **O (Big O)** — 渐进上界：`f(n) ≤ c·g(n)` 对所有 `n ≥ n₀`
- **Ω (Big Omega)** — 渐进下界：`f(n) ≥ c·g(n)` 对所有 `n ≥ n₀`
- **Θ (Big Theta)** — 紧渐进界：`c₁·g(n) ≤ f(n) ≤ c₂·g(n)` 对所有 `n ≥ n₀`

### 1.4 摊还分析

对一系列操作求平均时间复杂度（如动态数组的 `append` 偶尔 O(n) 扩容，摊还后 O(1)）。

### 1.5 空间复杂度

算法运行所需的额外存储空间（输入占用不计）。

|------|-----------|
| 随机访问 | O(1) |
| 末尾插入/删除 | O(1) |

|------|------|

|------|---------------------|

- 可用数组或链表实现

## 三、排序算法

### 3.1 冒泡排序

```python
def bubble_sort(arr):
    n = len(arr)
    # 外层循环：共进行 n-1 轮冒泡
    for i in range(n - 1):
        swapped = False
        # 内层循环：每轮将 [0, n-1-i] 区间内的最大值冒泡到最后
        for j in range(n - 1 - i):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        # 如果一轮下来没有交换，说明已经有序，提前终止
        if not swapped:
            break
```

**思路**：相邻元素两两比较，大的往后移，每一轮确定一个最大值放到末尾。优化：若某轮无交换则提前结束。

- 最好 O(n)（已有序），最坏 O(n²)，平均 O(n²)

### 3.2 选择排序

```python
def selection_sort(arr):
    n = len(arr)
    for i in range(n - 1):
        # 在 [i, n-1] 中找到最小元素的下标
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        # 将最小元素交换到位置 i
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
```

**思路**：每一轮从未排序部分选最小值，放到已排序部分的末尾。无论数据如何，比较次数固定。

- 始终 O(n²)，不稳定

### 3.3 插入排序

```python
def insertion_sort(arr):
    n = len(arr)
    # 从第 1 个元素开始（第 0 个视为已排序）
    for i in range(1, n):
        key = arr[i]
        # 将 key 插入到已排序部分的正确位置
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]  # 元素后移
            j -= 1
        arr[j + 1] = key
```

**思路**：将数组分为已排序（左边）和未排序（右边）两部分，每次从右边取一个元素插入到左边的正确位置。数据基本有序时效率高。

- 最好 O(n)（已有序），最坏 O(n²)，平均 O(n²)

### 3.4 归并排序 — 分治思想

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    # 1. 分：将数组从中间切分为左右两部分
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    # 2. 治：将两个有序子数组合并为一个有序数组
    return merge(left, right)

def merge(left, right):
    i = j = 0
    result = []
    # 双指针比较，将较小的元素依次放入结果数组
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    # 剩余元素直接追加（左右最多一边有剩余）
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

**思路**：分治法的经典应用。① 将数组递归地对半切分，直到每个子数组只有一个元素（天然有序）；② 回溯时两两合并，合并时用双指针比较并将较小者放入结果。

- 始终 O(n log n)，空间 O(n)

### 3.5 快速排序 — 分治思想

```python
def quick_sort(arr, low, high):
    if low < high:
        # 划分：选定 pivot，将数组分为小于 pivot 和大于 pivot 两部分
        pi = partition(arr, low, high)
        # 递归排序左右两部分
        quick_sort(arr, low, pi - 1)
        quick_sort(arr, pi + 1, high)

def partition(arr, low, high):
    # 选最后一个元素作为基准 pivot
    pivot = arr[high]
    i = low  # i 指向小于 pivot 区域的右边界
    for j in range(low, high):
        if arr[j] < pivot:
            arr[i], arr[j] = arr[j], arr[i]
            i += 1
    # 将 pivot 放到正确位置
    arr[i], arr[high] = arr[high], arr[i]
    return i
```

**思路**：① 选一个基准 pivot；② 一趟划分将数组分成两部分，左边全 ≤ pivot，右边全 ≥ pivot；③ 递归处理左右子数组。划分的核心是维护一个指针 `i`，它左边全是小于 pivot 的元素。

- 平均 O(n log n)，最坏 O(n²)（已有序且选最左/最右为 pivot）
- 优化：随机选 pivot / 三数取中法

### 3.6 堆排序

```python
def heapify(arr, n, i):
    # 将以 i 为根的子树堆化（下沉操作）
    largest = i          # 假设当前根最大
    left = 2 * i + 1     # 左子节点
    right = 2 * i + 2    # 右子节点

    # 找出 i、left、right 三者中的最大值
    if left < n and arr[left] > arr[largest]:
        largest = left
    if right < n and arr[right] > arr[largest]:
        largest = right
    # 如果最大值不是 i，交换并递归下沉
    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        heapify(arr, n, largest)

def heap_sort(arr):
    n = len(arr)
    # 建堆：从最后一个非叶子节点开始，自底向上堆化
    # 最后一个非叶子节点下标 = n//2 - 1
    for i in range(n // 2 - 1, -1, -1):
        heapify(arr, n, i)
    # 排序：每次将堆顶（最大值）交换到末尾，再对剩余部分堆化
    for i in range(n - 1, 0, -1):
        arr[0], arr[i] = arr[i], arr[0]  # 将堆顶放到末尾
        heapify(arr, i, 0)                # 对 [0, i-1] 范围堆化
```

**思路**：① 建堆（O(n)）：从最后一个非叶子节点开始下沉，使数组满足堆性质；② 排序（O(n log n)）：反复将堆顶（最值）交换到数组末尾，然后对剩余元素堆化。

- 始终 O(n log n)，不稳定，就地排序

## 四、树与树算法

### 4.1 基本术语

- **根节点**：没有父节点的节点
- **叶子节点**：没有子节点的节点
- **深度**：从根到某节点的边数
- **高度**：从某节点到最远叶子节点的边数
- **满二叉树**：所有非叶子节点都有两个子节点
- **完全二叉树**：除最后一层外全部填满，最后一层节点靠左排列

### 4.2 二叉树定义与遍历

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right
```

#### 递归遍历

```python
# 前序：根 -> 左 -> 右
def preorder(root):
    if not root:
        return
    print(root.val)       # 访问根
    preorder(root.left)   # 左子树
    preorder(root.right)  # 右子树

# 中序：左 -> 根 -> 右（在 BST 中输出升序序列）
def inorder(root):
    if not root:
        return
    inorder(root.left)
    print(root.val)
    inorder(root.right)

# 后序：左 -> 右 -> 根
def postorder(root):
    if not root:
        return
    postorder(root.left)
    postorder(root.right)
    print(root.val)
```

#### 非递归遍历（显式栈模拟）

```python
# 前序非递归：访问节点后将右孩子入栈，再左孩子入栈
def preorder_iter(root):
    if not root:
        return
    stack = [root]
    while stack:
        node = stack.pop()
        print(node.val)
        # 先右后左入栈，保证出栈时先处理左子树
        if node.right:
            stack.append(node.right)
        if node.left:
            stack.append(node.left)

# 中序非递归：一直向左走到底，沿途入栈，出栈时转向右子树
def inorder_iter(root):
    stack = []
    cur = root
    while cur or stack:
        # 一直向左走到最左
        while cur:
            stack.append(cur)
            cur = cur.left
        # 出栈并访问
        cur = stack.pop()
        print(cur.val)
        # 转向右子树
        cur = cur.right

# 后序非递归：双栈法或单栈 + prev 标记
def postorder_iter(root):
    if not root:
        return
    stack = [root]
    output = []
    while stack:
        node = stack.pop()
        output.append(node.val)
        # 前序是 根-左-右，后序调整为 左-右-根
        # 先左后右入栈，出栈顺序为 根-右-左，最后反转得到 左-右-根
        if node.left:
            stack.append(node.left)
        if node.right:
            stack.append(node.right)
    print(output[::-1])
```

### 4.3 二叉搜索树 (BST)

**性质**：左子树所有节点 < 根 < 右子树所有节点，中序遍历为严格升序。

```python
class BST:
    class _Node:
        def __init__(self, val):
            self.val = val
            self.left = None
            self.right = None

    def __init__(self):
        self.root = None

    # 查找 —— 类似二分查找的树形版本，O(log n) 平均
    def search(self, val):
        cur = self.root
        while cur:
            if val == cur.val:
                return True
            elif val < cur.val:
                cur = cur.left    # 小于根，去左子树找
            else:
                cur = cur.right   # 大于根，去右子树找
        return False

    # 插入 —— 先查找位置，再挂接新叶子节点
    def insert(self, val):
        if not self.root:
            self.root = self._Node(val)
            return
        cur = self.root
        while True:
            if val < cur.val:
                if cur.left:
                    cur = cur.left
                else:
                    cur.left = self._Node(val)
                    break
            else:
                if cur.right:
                    cur = cur.right
                else:
                    cur.right = self._Node(val)
                    break

    # 删除 —— 需要处理三种情况
    def delete(self, val):
        # 先找到待删除节点及其父节点
        parent, cur = None, self.root
        while cur and cur.val != val:
            parent = cur
            if val < cur.val:
                cur = cur.left
            else:
                cur = cur.right
        if not cur:
            return  # 没找到

        # 情况 1：叶子节点（无子节点）—— 直接从父节点删除
        # 情况 2：只有一个子节点 —— 子节点顶替待删节点
        if not cur.left or not cur.right:
            child = cur.left if cur.left else cur.right
            if not parent:          # 删除的是根节点
                self.root = child
            elif parent.left == cur:
                parent.left = child
            else:
                parent.right = child
        else:
            # 情况 3：有两个子节点
            # 找到右子树的最小节点（或左子树的最大节点）来替换
            successor = cur.right
            successor_parent = cur
            while successor.left:
                successor_parent = successor
                successor = successor.left
            # 用后继节点的值覆盖待删节点
            cur.val = successor.val
            # 删除后继节点（它一定没有左子节点，属于情况 1 或 2）
            if successor_parent.left == successor:
                successor_parent.left = successor.right
            else:
                successor_parent.right = successor.right
```

**删除的三种情况详解**：
1. **叶子节点**（无子节点）：直接断开父节点指向它的引用即可。
2. **有一个子节点**：让父节点直接指向该子节点，相当于跳过待删节点。
3. **有两个子节点**：找到右子树中最小的节点（后继），将其值复制到待删节点，然后删除后继节点。后继节点一定没有左子节点，因此转化为情况 1 或 2。

### 4.4 AVL 平衡二叉树

**平衡条件**：每个节点的左右子树高度差（平衡因子）为 -1、0 或 1。

```python
class AVLNode:
    def __init__(self, val):
        self.val = val
        self.left = None
        self.right = None
        self.height = 1  # 树高（用于计算平衡因子）

def get_height(node):
    return node.height if node else 0

def get_balance(node):
    # 平衡因子 = 左子树高 - 右子树高
    return get_height(node.left) - get_height(node.right)

# 右旋（LL 失衡）
#      y            x
#     / \          / \
#    x   T3  =>  T1  y
#   / \            / \
#  T1 T2          T2 T3
def right_rotate(y):
    x = y.left
    T2 = x.right
    # 旋转
    x.right = y
    y.left = T2
    # 更新高度（先更新 y 再更新 x）
    y.height = 1 + max(get_height(y.left), get_height(y.right))
    x.height = 1 + max(get_height(x.left), get_height(x.right))
    return x  # 新的根节点

# 左旋（RR 失衡）
def left_rotate(x):
    y = x.right
    T2 = y.left
    y.left = x
    x.right = T2
    x.height = 1 + max(get_height(x.left), get_height(x.right))
    y.height = 1 + max(get_height(y.left), get_height(y.right))
    return y

# 插入
def insert(node, val):
    if not node:
        return AVLNode(val)
    # 1. 按 BST 规则插入
    if val < node.val:
        node.left = insert(node.left, val)
    else:
        node.right = insert(node.right, val)

    # 2. 更新高度
    node.height = 1 + max(get_height(node.left), get_height(node.right))

    # 3. 检查平衡并旋转
    balance = get_balance(node)

    # LL 失衡：左孩子的左子树插入 -> 右旋
    if balance > 1 and val < node.left.val:
        return right_rotate(node)

    # RR 失衡：右孩子的右子树插入 -> 左旋
    if balance < -1 and val > node.right.val:
        return left_rotate(node)

    # LR 失衡：左孩子的右子树插入 -> 先左旋再右旋
    if balance > 1 and val > node.left.val:
        node.left = left_rotate(node.left)
        return right_rotate(node)

    # RL 失衡：右孩子的左子树插入 -> 先右旋再左旋
    if balance < -1 and val < node.right.val:
        node.right = right_rotate(node.right)
        return left_rotate(node)

    return node
```

**四种失衡类型速记**：
- **LL**（左左）：左孩子的左子树高了 → 右旋一次
- **RR**（右右）：右孩子的右子树高了 → 左旋一次
- **LR**（左右）：左孩子的右子树高了 → 先左旋再右旋
- **RL**（右左）：右孩子的左子树高了 → 先右旋再左旋

### 4.5 堆与优先队列

堆是一棵完全二叉树，用数组存储。下标 `i` 的父节点为 `(i-1)//2`，左子为 `2i+1`，右子为 `2i+2`。

**最小堆**：父节点 ≤ 子节点。Python 的 `heapq` 实现的就是最小堆。

```python
class MinHeap:
    def __init__(self):
        self.heap = []

    def _sift_up(self, idx):
        # 将 idx 处的元素上浮到正确位置（用于插入）
        # 不断与父节点比较，若小于父节点则上浮
        while idx > 0:
            parent = (idx - 1) // 2
            if self.heap[idx] >= self.heap[parent]:
                break
            self.heap[idx], self.heap[parent] = self.heap[parent], self.heap[idx]
            idx = parent

    def _sift_down(self, idx):
        # 将 idx 处的元素下沉到正确位置（用于 pop 和 heapify）
        n = len(self.heap)
        while True:
            smallest = idx
            left = 2 * idx + 1
            right = 2 * idx + 2
            if left < n and self.heap[left] < self.heap[smallest]:
                smallest = left
            if right < n and self.heap[right] < self.heap[smallest]:
                smallest = right
            if smallest == idx:
                break
            self.heap[idx], self.heap[smallest] = self.heap[smallest], self.heap[idx]
            idx = smallest

    def push(self, val):
        # 插入到末尾，然后上浮
        self.heap.append(val)
        self._sift_up(len(self.heap) - 1)

    def pop(self):
        # 弹出堆顶（最小值），将末尾元素换到堆顶，然后下沉
        if not self.heap:
            return None
        val = self.heap[0]
        last = self.heap.pop()
        if self.heap:
            self.heap[0] = last
            self._sift_down(0)
        return val

    @classmethod
    def heapify(cls, arr):
        # 从最后一个非叶子节点开始，自底向上下沉
        # 时间复杂度 O(n) —— Floyd 建堆算法
        h = cls()
        h.heap = arr[:]
        n = len(arr)
        for i in range(n // 2 - 1, -1, -1):
            h._sift_down(i)
        return h
```

**Floyd 建堆为 O(n) 而非 O(n log n) 的原因**：

建堆时从最后一个非叶子节点开始下沉，每个节点下沉的次数与其高度成正比。总工作量 = 每层节点数 × 下沉层数之和：

```
第 k 层（从底往上数 0,1,2,...） 有 n/2^(k+1) 个节点，最多下沉 k 步
总工作量 = Σ_{k=0}^{log n} n/2^(k+1) × k < n × Σ_{k=0}^{∞} k/2^k = O(n)
```

这个级数是收敛的，总和为 O(n)。

### 4.6 并查集 (Union-Find)

```python
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))  # 初始时每个节点的父节点是自己
        self.rank = [1] * n           # 树的秩（近似高度）

    def find(self, x):
        # 查找根节点（代表元）
        # 路径压缩：将路径上所有节点直接指向根
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, a, b):
        ra, rb = self.find(a), self.find(b)
        if ra == rb:
            return False  # 已经在同一集合
        # 按秩合并：将矮树合并到高树上，保持树高尽可能小
        if self.rank[ra] < self.rank[rb]:
            self.parent[ra] = rb
        elif self.rank[ra] > self.rank[rb]:
            self.parent[rb] = ra
        else:
            self.parent[rb] = ra
            self.rank[ra] += 1
        return True
```

**为什么近乎常数**：路径压缩 + 按秩合并使得单次操作的摊还复杂度为 O(α(n))，α(n) 是反阿克曼函数，对于所有实用规模的 n（甚至 n = 宇宙原子数），α(n) ≤ 5。

### 4.7 线段树 (Segment Tree) — 区间求和为例

```python
class SegmentTree:
    def __init__(self, arr):
        n = len(arr)
        # 线段树用数组存储，大小通常为 4n
        self.tree = [0] * (4 * n)
        self.n = n
        if n > 0:
            self._build(arr, 1, 0, n - 1)

    def _build(self, arr, node, l, r):
        # node: 当前线段树节点下标
        # [l, r]: 该节点代表的区间范围
        if l == r:
            # 叶子节点直接存储原数组值
            self.tree[node] = arr[l]
            return
        mid = (l + r) // 2
        self._build(arr, node * 2, l, mid)
        self._build(arr, node * 2 + 1, mid + 1, r)
        # 内部节点存储左右子节点的和
        self.tree[node] = self.tree[node * 2] + self.tree[node * 2 + 1]

    def query(self, ql, qr):
        # 查询区间 [ql, qr] 的和
        return self._query(1, 0, self.n - 1, ql, qr)

    def _query(self, node, l, r, ql, qr):
        # 三种情况：
        # 1. 当前区间完全在查询范围内 → 直接返回
        # 2. 当前区间与查询范围无交集 → 返回 0
        # 3. 有部分交集 → 递归查询左右子树
        if ql <= l and r <= qr:
            return self.tree[node]
        if r < ql or l > qr:
            return 0
        mid = (l + r) // 2
        left_sum = self._query(node * 2, l, mid, ql, qr)
        right_sum = self._query(node * 2 + 1, mid + 1, r, ql, qr)
        return left_sum + right_sum

    def update(self, idx, val):
        # 单点更新：将 arr[idx] 修改为 val
        self._update(1, 0, self.n - 1, idx, val)

    def _update(self, node, l, r, idx, val):
        if l == r:
            self.tree[node] = val
            return
        mid = (l + r) // 2
        if idx <= mid:
            self._update(node * 2, l, mid, idx, val)
        else:
            self._update(node * 2 + 1, mid + 1, r, idx, val)
        self.tree[node] = self.tree[node * 2] + self.tree[node * 2 + 1]
```

#### 懒惰标记（Lazy Propagation，区间更新）

当需要区间更新（如将 [L, R] 所有元素都加 x）时，如果每次都更新到叶子节点，最坏为 O(n)。引入 lazy 标记：更新时只更新完全被覆盖的节点，标记"欠账"，查询到该节点时才下推标记。

```python
class LazySegmentTree:
    def __init__(self, arr):
        n = len(arr)
        self.tree = [0] * (4 * n)
        self.lazy = [0] * (4 * n)  # 懒惰标记，表示"该节点的子节点还没被更新"
        self.n = n
        if n > 0:
            self._build(arr, 1, 0, n - 1)

    def _push(self, node, l, r):
        # 将 lazy 标记下推到子节点
        if self.lazy[node]:
            mid = (l + r) // 2
            # 左右子节点接收标记
            self.tree[node * 2] += self.lazy[node] * (mid - l + 1)
            self.tree[node * 2 + 1] += self.lazy[node] * (r - mid)
            self.lazy[node * 2] += self.lazy[node]
            self.lazy[node * 2 + 1] += self.lazy[node]
            self.lazy[node] = 0

    def range_add(self, ql, qr, val):
        # 将区间 [ql, qr] 的所有元素加 val
        self._range_add(1, 0, self.n - 1, ql, qr, val)

    def _range_add(self, node, l, r, ql, qr, val):
        if ql <= l and r <= qr:
            # 完全覆盖：更新当前节点并打上 lazy 标记
            self.tree[node] += val * (r - l + 1)
            self.lazy[node] += val
            return
        self._push(node, l, r)  # 下推标记
        mid = (l + r) // 2
        if ql <= mid:
            self._range_add(node * 2, l, mid, ql, qr, val)
        if qr > mid:
            self._range_add(node * 2 + 1, mid + 1, r, ql, qr, val)
        self.tree[node] = self.tree[node * 2] + self.tree[node * 2 + 1]
```

### 4.8 树状数组 (Fenwick Tree / BIT)

比线段树更轻量（省一半空间），但只能处理**前缀和**类问题。核心操作利用了 `lowbit = x & -x`。

```python
class BIT:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 1)  # 1-indexed

    def add(self, idx, delta):
        # 在位置 idx 增加 delta，同时更新所有相关节点
        # 从 idx 开始，每次 i += lowbit(i)，直到 n
        i = idx
        while i <= self.n:
            self.bit[i] += delta
            i += i & -i

    def prefix_sum(self, idx):
        # 求 [1, idx] 的前缀和
        # 从 idx 开始，每次 i -= lowbit(i)，直到 0
        res = 0
        i = idx
        while i > 0:
            res += self.bit[i]
            i -= i & -i
        return res

    def range_sum(self, l, r):
        # 利用前缀和求区间和
        return self.prefix_sum(r) - self.prefix_sum(l - 1)
```

**lowbit 的含义**：`i & -i` 表示 `i` 的二进制中最低位的 1 所代表的数值。

**BIT 与线段树的对比**：
- BIT 只能处理可差分的信息（如和、异或），不能处理区间最大值/最小值
- BIT 代码短、常数小、空间省一半
- 如果需要区间最大值、或者需要区间更新 + 区间查询的复杂操作，用线段树

## 五、图与图算法

### 5.1 基本概念

- **有向图 / 无向图**：边是否有方向
- **加权图**：边带权值
- **连通图 / 强连通图**：无向图任意两点可达 / 有向图任意两点互相可达
- **度**：无向图中节点的邻居数；有向图中分为入度和出度

### 5.2 存储方式

| 方式 | 空间 | 适用场景 |
|------|------|---------|
| 邻接矩阵 | O(V²) | 稠密图，快速判断边是否存在 |
| 邻接表 | O(V+E) | 稀疏图，遍历邻居高效 |

# 邻接表（有向图示例）
# graph[u] = [(v, weight), ...]
graph = {

### 5.3 图的遍历

**DFS — 深度优先搜索**：

```python
def dfs(graph, start):
    visited = set()

    def _dfs(u):
        visited.add(u)
        print(f"访问节点 {u}")
        for v in graph[u]:
            if v not in visited:
                _dfs(v)
    _dfs(start)


# 非递归版（显式栈）
def dfs_iter(graph, start):
    visited = set()
    stack = [start]
    while stack:
        u = stack.pop()
        if u in visited:
            continue
        visited.add(u)
        print(f"访问节点 {u}")
        # 逆序入栈以保持与递归版一致的访问顺序
        for v in reversed(graph[u]):
            if v not in visited:
                stack.append(v)
```

**BFS — 广度优先搜索**：

```python
from collections import deque

def bfs(graph, start):
    visited = {start}
    queue = deque([start])
    while queue:
        u = queue.popleft()
        print(f"访问节点 {u}")
        for v in graph[u]:
            if v not in visited:
                visited.add(v)
                queue.append(v)
```

**DFS 与 BFS 的核心区别**：
- DFS 使用**栈**（递归本质也是栈），沿着一条路径走到黑再回溯
- BFS 使用**队列**，按距离起点从近到远逐层扩展
- BFS 在无权图中第一次访问到某节点时，路径一定是最短的

### 5.4 最短路径

#### 5.4.1 Dijkstra 算法（单源，非负权）

**核心思想**：贪心。维护一个"未确定最短距离"的集合，每次从集合中选距离最小的节点 u，然后用 u 去松弛它的邻居（即尝试通过 u 缩短起点到邻居的距离）。

```python
import heapq

def dijkstra(graph, n, start):
    # dist[i] 表示从 start 到节点 i 的当前最短距离
    dist = [float('inf')] * n
    dist[start] = 0

    # 优先队列存储 (距离, 节点)，每次弹出距离最小的节点
    pq = [(0, start)]

    while pq:
        d, u = heapq.heappop(pq)

        # 重要优化：如果弹出的距离大于已记录的最短距离，跳过
        # （因为同一节点可能被多次入队）
        if d > dist[u]:
            continue

        # 松弛操作：尝试通过 u 改善邻居 v 的距离
        for v, w in graph[u]:
            new_dist = dist[u] + w
            if new_dist < dist[v]:
                dist[v] = new_dist
                heapq.heappush(pq, (new_dist, v))

    return dist
```

**正确性证明关键**：非负权保证每次从优先队列中弹出的节点，其当前距离已经是最终最短距离。因为所有边权非负，任何后续找到的路径都不会比当前更短。

**时间复杂度**：O((V + E) log V) — 每个节点最多入队出队一次，每条边检查一次。

#### 5.4.2 Bellman-Ford 算法（含负权）

**核心思想**：对全部 V-1 轮松弛，每轮对所有边尝试松弛。如果第 V 轮还能松弛，说明存在负环。

```python
def bellman_ford(edges, n, start):
    # edges = [(u, v, w), ...]
    dist = [float('inf')] * n
    dist[start] = 0

    # 第 1 到 V-1 轮松弛
    # 为什么 V-1 轮？因为最短路径最多包含 V-1 条边（无环情况）
    for _ in range(n - 1):
        updated = False
        for u, v, w in edges:
            if dist[u] != float('inf') and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                updated = True
        # 优化：如果某一轮没有更新，说明已经收敛，提前退出
        if not updated:
            break

    # 第 V 轮检测负环
    for u, v, w in edges:
        if dist[u] != float('inf') and dist[u] + w < dist[v]:
            return None  # 存在负环，无解

    return dist
```

**关于为什么是 V-1 轮**：在一个没有负环的图中，最短路径不会包含环（因为如果是正环可以去掉缩短距离，零环不影响，负环导致路径无限短）。因此最短路径最多有 V-1 条边，经过 V-1 轮松弛一定能得到最短距离。在第 V 轮如果还能松弛，说明存在负环。

#### 5.4.3 Floyd-Warshall 算法（全源最短路）

**核心思想**：动态规划。`dp[k][i][j]` 表示从 i 到 j 的路径中，中间节点只经过编号 ≤ k 的节点时的最短距离。

滚动数组优化后变为二维：

```python
def floyd_warshall(n, graph):
    # graph 为邻接矩阵，graph[i][j] = i 到 j 的边权，不存在为 inf
    # 注意：graph[i][i] = 0
    dist = [row[:] for row in graph]  # 拷贝

    # k 是中间节点集合的上限：允许路径经过节点 0..k
    for k in range(n):
        # i 是起点，j 是终点
        for i in range(n):
            # 剪枝：dist[i][k] 已经是 inf，不可能通过 k 更新
            if dist[i][k] == float('inf'):
                continue
            for j in range(n):
                # 状态转移：dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
                if dist[i][k] + dist[k][j] < dist[i][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]

    return dist
```

**为啥 k 在最外层**：`dist[i][j]` 的更新依赖 `dist[i][k]` 和 `dist[k][j]`，k 必须在最外层确保第 k 轮使用的 `dist` 值是基于 k-1 轮的，即中间节点集合最多为 0..k-1 的结果。

**检测负环**：如果某个 `dist[i][i] < 0`，说明存在负环。

**时间复杂度**：O(V³)，适合稠密图或 V ≤ 500 的情况。

### 5.5 最小生成树 (MST)

#### 5.5.1 Kruskal 算法

**核心思想**：贪心。将所有边按权值从小到大排序，依次加入，加入时用并查集判断是否形成环。

```python
def kruskal(n, edges):
    # edges = [(u, v, w), ...]
    # 1. 按权值排序
    edges.sort(key=lambda x: x[2])

    uf = UnionFind(n)
    total_weight = 0
    mst_edges = []

    # 2. 从小到大遍历边
    for u, v, w in edges:
        # 如果 u 和 v 不在同一集合，加入这条边不会形成环
        if uf.union(u, v):
            total_weight += w
            mst_edges.append((u, v, w))
            # 如果已经收集了 n-1 条边，MST 构建完成
            if len(mst_edges) == n - 1:
                break

    return total_weight, mst_edges
```

**正确性证明**（切分定理）：对于任意一个边权不重复的图，Kruskal 每次选择的边都是连接两个不同连通分量的最小边，一定属于某个 MST。

#### 5.5.2 Prim 算法

**核心思想**：从任意节点开始，维护一个已选节点集合 S，每次选择连接 S 与剩余节点的最小边，将对应节点加入 S。实现方式与 Dijkstra 几乎一样，区别在于 Prim 用距离表示"到已选集合的最短距离"而非"到起点的最短距离"。

```python
import heapq

def prim(graph, n, start=0):
    # graph[u] = [(v, w), ...]
    visited = [False] * n
    # 优先队列存储 (权值, 节点, 来源节点)
    pq = [(0, start, -1)]
    total = 0
    mst_edges = []

    while pq:
        w, u, src = heapq.heappop(pq)
        if visited[u]:
            continue
        visited[u] = True
        total += w
        if src != -1:
            mst_edges.append((src, u, w))

        for v, w2 in graph[u]:
            if not visited[v]:
                heapq.heappush(pq, (w2, v, u))

    return total, mst_edges
```

**Prim vs Kruskal**：
- Kruskal：O(E log E)，适合稀疏图（E 较小）
- Prim（堆）：O(E log V)，适合稠密图（E 接近 V²）
- Kruskal 实现简单，需要并查集；Prim 需要优先队列

### 5.6 拓扑排序 (Topological Sort)

应用于**有向无环图 (DAG)**，将图中所有节点排成一个线性序列，使得对每条边 (u, v)，u 在序列中出现在 v 之前。

#### Kahn 算法（BFS 删入度法）

```python
from collections import deque

def topological_sort_kahn(n, graph):
    # 1. 计算所有节点的入度
    indegree = [0] * n
    for u in range(n):
        for v in graph[u]:
            indegree[v] += 1

    # 2. 将所有入度为 0 的节点入队
    #    入度为 0 意味着该节点没有前驱依赖
    queue = deque([u for u in range(n) if indegree[u] == 0])
    result = []

    while queue:
        u = queue.popleft()
        result.append(u)
        # 删除节点 u：将其所有邻居的入度减 1
        for v in graph[u]:
            indegree[v] -= 1
            # 如果邻居入度变为 0，说明其所有前驱都已处理，可以入队
            if indegree[v] == 0:
                queue.append(v)

    # 如果 result 长度 < n，说明图中有环
    if len(result) != n:
        return None  # 存在环，无法拓扑排序
    return result
```

#### DFS 后序逆序法

```python
def topological_sort_dfs(n, graph):
    visited = [0] * n  # 0: 未访问, 1: 访问中, 2: 已结束
    result = []
    has_cycle = False

    def dfs(u):
        nonlocal has_cycle
        visited[u] = 1  # 标记为"访问中"
        for v in graph[u]:
            if visited[v] == 1:
                # 发现回边，说明存在环
                has_cycle = True
                return
            if visited[v] == 0:
                dfs(v)
        visited[u] = 2   # 标记为"已结束"
        result.append(u)  # 后序加入结果

    for u in range(n):
        if visited[u] == 0:
            dfs(u)
            if has_cycle:
                return None

    # 后序结果的逆序即为拓扑序
    return result[::-1]
```

### 5.7 强连通分量 (SCC) — 识别与分离

**强连通分量**：在有向图中，如果两个节点 u、v 互相可达（u → v 和 v → u 均有路径），则称 u 和 v 强连通。将图中所有节点按强连通关系分组，每组是一个极大强连通子图，即 SCC。

**核心性质**：将每个 SCC 缩成一个点后，原图变为**有向无环图 (DAG)**。这个 DAG 称为 **SCC 缩点图（ condensation graph）**，是解决许多有向图问题（如 2-SAT、差分约束、路径计数）的关键中间步骤。

---

#### 5.7.1 Tarjan 算法

**核心思想**：DFS 过程中为每个节点记录两个时间戳，通过栈维护当前路径上的节点，当发现一个 SCC 的根时弹出整个分量。

```
dfn[u] = 节点 u 被发现的时间（发现顺序编号，全局唯一）
low[u] = 从 u 出发，通过子树或一条回边（不能经过横叉边）
         能追溯到的 dfn 最小的节点的时间戳
```

当 `dfn[u] == low[u]` 时，u 是一个 SCC 的根。此时从 u 到**栈顶**的所有节点构成一个 SCC（因为它们是 u 的子树中尚未被分配 SCC 的节点）。

```python
def tarjan_scc(n, graph):
    """
    Tarjan 算法求有向图的强连通分量
    参数:
        n: 节点数 (0 ~ n-1)
        graph: 邻接表，graph[u] = [v, ...]
    返回:
        scc_count: SCC 的数量
        scc_id: 每个节点所属的 SCC 编号 (1-indexed)
    """
    dfn = [0] * n              # 发现时间戳，0 表示未访问
    low = [0] * n              # 最早可达节点的时间戳
    timestamp = 0
    stack = []                 # 维护当前 DFS 路径上尚未分配 SCC 的节点
    in_stack = [False] * n
    scc_id = [-1] * n          # 每个节点所属 SCC 编号
    scc_nodes = []             # scc_nodes[i] = 第 i 个 SCC 包含的节点列表
    scc_count = 0

    def dfs(u):
        nonlocal timestamp, scc_count
        timestamp += 1
        dfn[u] = low[u] = timestamp
        stack.append(u)
        in_stack[u] = True

        # 遍历所有邻居
        for v in graph[u]:
            if dfn[v] == 0:
                # 情况 1：树边（v 未被访问）
                # 递归访问 v，然后用 v 的 low 值更新 u 的 low 值
                dfs(v)
                low[u] = min(low[u], low[v])

            elif in_stack[v]:
                # 情况 2：回边（v 在栈中，即 v 是 u 的祖先或同路径节点）
                # 用 dfn[v] 更新 low[u]，表示 u 能绕回 v
                low[u] = min(low[u], dfn[v])

            # 情况 3：横叉边（v 已访问且不在栈中）
            # v 属于另一个已经处理完成的 SCC，直接忽略

        # 如果 dfn[u] == low[u]，说明 u 是一个 SCC 的根
        if dfn[u] == low[u]:
            scc_count += 1
            comp = []
            while True:
                w = stack.pop()
                in_stack[w] = False
                scc_id[w] = scc_count
                comp.append(w)
                if w == u:
                    break
            scc_nodes.append(comp)

    for u in range(n):
        if dfn[u] == 0:
            dfs(u)

    return scc_count, scc_id, scc_nodes
```

**Tarjan 算法执行示例**：

考虑图：0 → 1 → 2 → 0, 1 → 3 → 4 → 3

```
DFS 访问顺序: 0 → 1 → 2 → 0(回边) → 回溯 → 3 → 4 → 3(回边)

节点  dfn  low  解析
0      1    1    dfn[2] = 3 更新 low[1] = min(1,3) = 1, 最终 dfn==low → SCC {0,1,2}
1      2    1    由 2 的回边更新 low[1] = min(2, dfn[0]=1) = 1
2      3    1    发现回边 2→0, low[2] = min(3, dfn[0]=1) = 1
3      4    4    子树处理完毕, dfn[3]==low[4]==4 → SCC {3,4}
4      5    4    发现回边 4→3, low[4] = min(5, dfn[3]=4) = 4
```

最终得到两个 SCC：{0,1,2} 和 {3,4}

**low[u] 更新规则总结**：

| 边的类型 | 条件 | 更新方式 | 含义 |
|---------|------|---------|------|
| 树边 | v 未访问 | `low[u] = min(low[u], low[v])` | u 可达 v 可达的所有节点 |
| 回边 | v 在栈中 | `low[u] = min(low[u], dfn[v])` | u 有一条边绕回祖先 v |
| 横叉边 | v 不在栈中 | 忽略 | v 的 SCC 已确定，不影响当前 SCC |

**为什么回边用 `dfn[v]` 而非 `low[v]`**：如果用 `low[v]` 更新回边，可能跨越已确定的 SCC 边界，导致多个 SCC 被错误合并为一个。因为 `low[v]` 可能已经回溯到了某个更早的 SCC 的根节点，而 v 并不在该 SCC 中。

#### 5.7.2 Kosaraju 算法

**核心思想**：利用"原图和反图具有相同的 SCC"这一性质，分两次 DFS。

1. 第一次 DFS 在原图上进行，记录节点的**完成时间（出栈顺序）**
2. 按出栈顺序的**逆序**在反图上进行第二次 DFS，每次遍历到的节点属于同一个 SCC

```python
def kosaraju_scc(n, graph):
    """
    Kosaraju 算法求有向图的强连通分量
    参数:
        n: 节点数 (0 ~ n-1)
        graph: 邻接表，graph[u] = [v, ...]
    返回:
        scc_count: SCC 的数量
        scc_id: 每个节点所属的 SCC 编号
        scc_nodes: 每个 SCC 包含的节点列表
    """
    # 1. 构建反图（边的方向全部反转）
    rev_graph = [[] for _ in range(n)]
    for u in range(n):
        for v in graph[u]:
            rev_graph[v].append(u)

    # 2. 第一次 DFS：在原图上记录完成顺序
    visited = [False] * n
    order = []  # 存储完成顺序（后序）

    def dfs1(u):
        visited[u] = True
        for v in graph[u]:
            if not visited[v]:
                dfs1(v)
        order.append(u)  # 后序：访问完所有邻接点后加入

    for u in range(n):
        if not visited[u]:
            dfs1(u)

    # 3. 第二次 DFS：在反图上按完成顺序逆序遍历
    scc_id = [-1] * n
    scc_nodes = []
    scc_count = 0

    def dfs2(u, cid):
        scc_id[u] = cid
        scc_nodes[-1].append(u)
        for v in rev_graph[u]:
            if scc_id[v] == -1:
                dfs2(v, cid)

    # 按完成顺序逆序遍历（order[-1], order[-2], ...）
    for u in reversed(order):
        if scc_id[u] == -1:
            scc_count += 1
            scc_nodes.append([])
            dfs2(u, scc_count)

    return scc_count, scc_id, scc_nodes
```

**为什么 Kosaraju 是正确的**：

- 第一次 DFS 的完成顺序保证了：如果 SCC A 在缩点图 DAG 中是 SCC B 的前驱（A → B），则 A 中某个节点的完成时间**晚于** B 中所有节点的完成时间（或者反过来，取决于实现）。
- 在反图上，原图 A → B 的关系变为 B → A。按完成顺序逆序遍历等价于从缩点图 DAG 的**源点（入度为 0 的 SCC）**开始处理，每次遍历恰好得到一个 SCC，不会"溢出"到其他 SCC。

**Tarjan vs Kosaraju 对比**：

| 维度 | Tarjan | Kosaraju |
|------|--------|----------|
| 时间复杂度 | O(V+E) | O(V+E) |
| DFS 次数 | 1 次 | 2 次 |
| 空间 | 栈 + dfn/low | visited + order + 反图 |
| 代码复杂 | 较复杂（需理解 low/dfn 含义） | 简单直观 |
| 适用场景 | 需要 low 值做更多分析（如桥、割点） | 仅需单纯求 SCC |
| 常数 | 小 | 稍大（需额外建反图） |

#### 5.7.3 SCC 缩点 (Condensation)

将每个 SCC 视为一个节点，原图中跨越不同 SCC 的边保留为缩点图 DAG 的边。

```python
def build_condensation(n, graph, scc_count, scc_id):
    """
    构建 SCC 缩点图（DAG）
    返回:
        cond_graph: 缩点后的邻接表（节点编号 1 ~ scc_count）
        cond_indegree: 缩点后每个节点的入度
    """
    cond_graph = [set() for _ in range(scc_count + 1)]  # 1-indexed
    cond_indegree = [0] * (scc_count + 1)

    for u in range(n):
        cid_u = scc_id[u]
        for v in graph[u]:
            cid_v = scc_id[v]
            if cid_u != cid_v:
                # 跨 SCC 的边，在缩点图中保留（用 set 去重）
                if cid_v not in cond_graph[cid_u]:
                    cond_graph[cid_u].add(cid_v)
                    cond_indegree[cid_v] += 1

    # 将 set 转为 list
    cond_graph = [list(s) for s in cond_graph]
    return cond_graph, cond_indegree
```

**缩点后的性质**：
- 缩点图一定是一个 **DAG**（有向无环图），因为如果有环则环上的所有 SCC 应合并为一个
- 缩点图上的拓扑序即为 SCC 的拓扑序
- 入度为 0 的 SCC 是 DAG 的源点，出度为 0 的 SCC 是 DAG 的汇点

**缩点的应用**：
- **2-SAT 问题**：每个 SCC 内的变量赋值一致，若 x 和 ¬x 在同一 SCC 内则无解
- **最小点基**：缩点后入度为 0 的 SCC 中每个至少选一个点
- **可达性统计**：缩点后 DAG 上 DP 统计可达节点数，用 bitset 优化
- **差分约束最长路**：缩点后如果同一 SCC 内有正权环则无解
- **HDU 2767 / POJ 1236**：加多少条边可使图变为强连通——设缩点后入度为 0 的节点数 a，出度为 0 的节点数 b，答案为 `max(a, b)`

def is_strongly_connected(n, graph):
    """判断有向图是否为强连通图（即整张图是一个 SCC）"""

#### 5.7.5 实例：2-SAT 可行性判定

**问题**：给定 n 个布尔变量和若干子句 `(a ∨ b)`，判断是否存在一组赋值使所有子句为真。

**建图方法**：每个变量拆为两个节点（真/假），每个子句 `(a ∨ b)` 等价于 `(¬a → b) ∧ (¬b → a)`。建图后求 SCC，若某个变量的真/假节点在同一 SCC 内则无解。

```python
def two_sat(n, clauses):
    """
    2-SAT 可行性判定
    参数:
        n: 变量个数 (编号 1 ~ n)
        clauses: [(a, b), ...] 表示子句 (a ∨ b)
                 正数表示变量 x，负数表示 ¬x
    返回:
        True/False 是否有解
        assignment: 若有解，每个变量的赋值 (True/False)
    """
    N = 2 * n + 1  # 节点总数，用 1~n 表示 x，n+1~2n 表示 ¬x
    graph = [[] for _ in range(2 * n + 1)]

    def node_id(x):
        """将文字映射为节点编号: x → id(x), ¬x → id(¬x)"""
        if x > 0:
            return x          # x
        else:
            return n - x      # ¬x (如 n=3, ¬1 → 4)

    def add_impl(u, v):
        """添加蕴含边 u → v"""
        graph[u].append(v)

    for a, b in clauses:
        # (a ∨ b) ≡ (¬a → b) ∧ (¬b → a)
        add_impl(node_id(-a), node_id(b))
        add_impl(node_id(-b), node_id(a))

    # 求 SCC
    scc_count, scc_id, _ = tarjan_scc(2 * n + 1, graph)

    # 检查是否存在 x 和 ¬x 在同一个 SCC 中
    assignment = [False] * (n + 1)
    for i in range(1, n + 1):
        if scc_id[i] == scc_id[i + n]:
            return False, None  # 无解
        # 赋值策略：选择拓扑序较大的节点（即 scc_id 较小者）
        # Tarjan 的 scc_id 分配顺序与拓扑序相反
        assignment[i] = (scc_id[i] < scc_id[i + n])

    return True, assignment
```

### 5.8 网络流 — Dinic 算法

**核心思想**：Edmonds-Karp 的优化，每次寻找增广路时先对残量网络做 BFS 分层（Level Graph），然后在分层图上用 DFS 一次性寻找多条增广路（多路增广）。

```python
from collections import deque

class Dinic:
    def __init__(self, n):
        self.n = n
        self.graph = [[] for _ in range(n)]  # 邻接表

    def add_edge(self, u, v, c):
        # 添加正向边和反向边（反向边初始容量为 0）
        # 用 index 将正向边和反向边配对：正向边为偶数，反向边为奇数
        self.graph[u].append([v, c, len(self.graph[v])])  # [to, capacity, rev_index]
        self.graph[v].append([u, 0, len(self.graph[u]) - 1])

    def bfs_level(self, s, t):
        # BFS 构建分层图，返回是否能到达汇点
        self.level = [-1] * self.n
        q = deque([s])
        self.level[s] = 0
        while q:
            u = q.popleft()
            for v, cap, rev in self.graph[u]:
                if cap > 0 and self.level[v] == -1:
                    self.level[v] = self.level[u] + 1
                    q.append(v)
        return self.level[t] != -1

    def dfs_flow(self, u, t, f):
        # 在分层图上 DFS，发送流量 f，返回实际发送的流量
        if u == t:
            return f
        # it 数组用于当前弧优化：已经处理过的边不再重复尝试
        for i in range(self.it[u], len(self.graph[u])):
            self.it[u] = i
            v, cap, rev = self.graph[u][i]
            if cap > 0 and self.level[v] == self.level[u] + 1:
                # 只走下一层的节点（保证不绕圈）
                pushed = self.dfs_flow(v, t, min(f, cap))
                if pushed > 0:
                    # 更新正向边和反向边容量
                    self.graph[u][i][1] -= pushed
                    self.graph[v][rev][1] += pushed
                    return pushed
        return 0

    def max_flow(self, s, t):
        flow = 0
        INF = 10 ** 9
        while self.bfs_level(s, t):       # 1. BFS 分层
            self.it = [0] * self.n         # 初始化当前弧指针
            while True:
                pushed = self.dfs_flow(s, t, INF)  # 2. DFS 多路增广
                if pushed == 0:
                    break
                flow += pushed
        return flow
```

**Dinic 的关键优势**：
1. **分层图**：BFS 保证每次 DFS 只沿最短的增广路径
2. **多路增广**：一个 DFS 可以找到并发送多条增广路
3. **当前弧优化**：记录每个节点已经处理到的边，避免重复遍历

**时间复杂度**：O(V²E)，对于单位容量图（如二分图匹配）为 O(E√V)。

### 6.1 分治 (Divide & Conquer)

三步：① 分（分解为子问题）；② 治（递归求解子问题）；③ 合（合并子问题结果）。

**经典应用：归并排序**（见排序章节）—— 分解到单元素，合并时双指针比较。

**经典应用：快速排序**（见排序章节）—— 选 pivot 划分，递归排序左右部分。

**经典应用：最大子数组和（分治版）**：

```python
def max_subarray_sum(arr, l, r):
    if l == r:
        return arr[l]
    mid = (l + r) // 2
    # 最大和在左边、右边、跨越中间这三种情况
    left_max = max_subarray_sum(arr, l, mid)
    right_max = max_subarray_sum(arr, mid + 1, r)

    # 跨越中间：从中间向左右两边扩展求最大和
    left_sum = 0
    cur = 0
    for i in range(mid, l - 1, -1):
        cur += arr[i]
        left_sum = max(left_sum, cur)

    right_sum = 0
    cur = 0
    for i in range(mid + 1, r + 1):
        cur += arr[i]
        right_sum = max(right_sum, cur)

    cross_max = left_sum + right_sum
    return max(left_max, right_max, cross_max)
```

时间复杂度 O(n log n)，Kadane 算法（DP）可做到 O(n)。

### 6.2 贪心 (Greedy)

**核心**：每步都选当前最优的选择，期望最终结果是全局最优。

**使用前提**：问题具有**最优子结构**且贪心选择性质成立。

**经典示例：区间调度 — 选最多不相交区间**：

```python
def interval_scheduling(intervals):
    # intervals = [(start, end), ...]
    # 贪心策略：按结束时间最早排序，优先选结束早的
    intervals.sort(key=lambda x: x[1])
    count = 0
    last_end = -float('inf')
    for start, end in intervals:
        if start >= last_end:
            count += 1
            last_end = end
    return count
```

**经典示例：Huffman 编码**：

```python
import heapq

def huffman_encode(freq):
    # freq = {'a': 5, 'b': 9, 'c': 12, 'd': 13, 'e': 16, 'f': 45}
    # 1. 将每个字符作为叶子节点放入最小堆
    heap = [[w, [char, ""]] for char, w in freq.items()]
    heapq.heapify(heap)

    # 2. 每次合并权值最小的两个节点
    while len(heap) > 1:
        lo = heapq.heappop(heap)   # 最小编码树
        hi = heapq.heappop(heap)   # 次小编码树
        # 左分支加 '0'，右分支加 '1'
        for pair in lo[1:]:
            pair[1] = '0' + pair[1]
        for pair in hi[1:]:
            pair[1] = '1' + pair[1]
        # 合并后放回堆
        heapq.heappush(heap, [lo[0] + hi[0]] + lo[1:] + hi[1:])

    return sorted(heap[0][1:], key=lambda p: (len(p[1]), p[0]))
```

### 6.3 动态规划 (Dynamic Programming, DP)

**核心**：无后效性 + 最优子结构 + 重叠子问题。

**解题五步**：
1. 定义状态（`dp[i]` / `dp[i][j]` 表示什么）
2. 确定状态转移方程
3. 初始化（边界条件）
4. 确定遍历顺序
5. 返回结果

#### 示例 1：0/1 背包

```python
def knapsack_01(weights, values, capacity):
    n = len(weights)
    # dp[c] = 容量为 c 的背包能装的最大价值
    dp = [0] * (capacity + 1)

    for i in range(n):
        # 逆序遍历容量，保证每个物品最多选一次
        # 如果正序遍历，dp[w - weights[i]] 可能已经包含了当前第 i 个物品，导致物品被重复选取
        for w in range(capacity, weights[i] - 1, -1):
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])

    return dp[capacity]
```

**为什么容量要逆序**：因为一维 DP 中 `dp[w]` 依赖于 `dp[w - weights[i]]`（上一行的值）。如果正序遍历，`dp[w - weights[i]]` 可能已经被当前物品更新过，导致同一物品被多次放入。

#### 示例 2：完全背包

```python
def knapsack_complete(weights, values, capacity):
    n = len(weights)
    dp = [0] * (capacity + 1)

    for i in range(n):
        # 正序遍历，允许同一物品被多次选取
        for w in range(weights[i], capacity + 1):
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])

    return dp[capacity]
```

**0/1 背包与完全背包的区别**：只在容量循环方向不同。逆序避免重复，正序允许重复。

#### 示例 3：最长公共子序列 (LCS)

```python
def longest_common_subsequence(s1, s2):
    m, n = len(s1), len(s2)
    # dp[i][j] = s1[:i] 和 s2[:j] 的 LCS 长度
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i - 1] == s2[j - 1]:
                # 字符相同，长度 +1
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                # 字符不同，取 max(去掉 s1 当前字符, 去掉 s2 当前字符)
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    # 回溯构造 LCS 字符串
    lcs = []
    i, j = m, n
    while i > 0 and j > 0:
        if s1[i - 1] == s2[j - 1]:
            lcs.append(s1[i - 1])
            i -= 1
            j -= 1
        elif dp[i - 1][j] > dp[i][j - 1]:
            i -= 1
        else:
            j -= 1

    return dp[m][n], ''.join(reversed(lcs))
```

#### 示例 4：最长递增子序列 (LIS) — O(n log n) 优化版

```python
import bisect

def length_of_LIS(nums):
    # tails[i] = 长度为 i+1 的递增子序列的末尾元素的最小值
    # tails 数组本身是严格递增的
    tails = []
    for x in nums:
        # 二分查找第一个 >= x 的位置
        i = bisect.bisect_left(tails, x)
        if i == len(tails):
            tails.append(x)   # x 比所有末尾元素都大，可以延长 LIS
        else:
            tails[i] = x       # 替换，使长度为 i+1 的 LIS 末尾尽可能小
    return len(tails)
```

**核心思想**：维护 tails 数组，`tails[i]` 表示长度为 i+1 的递增子序列的最小末尾值。对于每个 x，找到第一个 ≥ x 的 tails 元素并替换，保证后续更有可能构成长序列。

#### 示例 5：编辑距离 (Levenshtein Distance)

```python
def min_edit_distance(s1, s2):
    m, n = len(s1), len(s2)
    # dp[i][j] = s1[:i] 转化为 s2[:j] 的最小编辑距离
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    # 初始化边界
    for i in range(m + 1):
        dp[i][0] = i  # 删除 i 个字符
    for j in range(n + 1):
        dp[0][j] = j  # 插入 j 个字符

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i - 1] == s2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]  # 字符相同，无需操作
            else:
                dp[i][j] = min(
                    dp[i - 1][j] + 1,      # 删除 s1[i-1]
                    dp[i][j - 1] + 1,      # 插入 s2[j-1]
                    dp[i - 1][j - 1] + 1   # 替换 s1[i-1] 为 s2[j-1]
                )

    return dp[m][n]
```

### 6.4 回溯 (Backtracking)

**核心**：DFS + 剪枝。在搜索过程中尝试每一种可能，走不通时回退到上一步。

```python
# 示例：N 皇后问题
def solve_n_queens(n):
    cols = set()       # 已占用的列
    diag1 = set()      # 已占用的主对角线（行+列）
    diag2 = set()      # 已占用的副对角线（行-列）
    board = [['.'] * n for _ in range(n)]
    result = []

    def backtrack(row):
        if row == n:
            # 找到一个解，保存
            result.append([''.join(r) for r in board])
            return
        for col in range(n):
            # 剪枝：检查列和对角线是否被占用
            if col in cols or (row + col) in diag1 or (row - col) in diag2:
                continue
            # 做选择
            board[row][col] = 'Q'
            cols.add(col)
            diag1.add(row + col)
            diag2.add(row - col)
            # 进入下一行
            backtrack(row + 1)
            # 撤销选择
            board[row][col] = '.'
            cols.remove(col)
            diag1.remove(row + col)
            diag2.remove(row - col)

    backtrack(0)
    return result
```

```python
# 示例 2：骑士周游问题 (Knight's Tour)
# 问题：在 n×n 棋盘上，骑士从起点出发，恰好不重复地访问每个格子一次
# 关键优化：Warnsdorff 规则——每次选择下一步可选格数最少的格子

def knights_tour(n, start_row=0, start_col=0):
    """
    骑士周游问题求解（含 Warnsdorff 启发式优化）
    返回一个 n×n 矩阵，board[r][c] = 第几步到达该格，无解返回 None
    """
    # 骑士的 8 种走法
    moves = [
        (-2, -1), (-2, 1), (-1, -2), (-1, 2),
        (1, -2), (1, 2), (2, -1), (2, 1)
    ]
    board = [[-1] * n for _ in range(n)]

    def is_valid(r, c):
        return 0 <= r < n and 0 <= c < n and board[r][c] == -1

    def get_degree(r, c):
        """计算位置 (r, c) 的下一步可选格数（Warnsdorff 启发式）"""
        cnt = 0
        for dr, dc in moves:
            nr, nc = r + dr, c + dc
            if is_valid(nr, nc):
                cnt += 1
        return cnt

    def backtrack(r, c, step):
        board[r][c] = step

        if step == n * n - 1:
            return True  # 已访问所有格子

        # 收集所有下一步可选位置，按 Warnsdorff 规则排序（可选格数少的优先）
        next_positions = []
        for dr, dc in moves:
            nr, nc = r + dr, c + dc
            if is_valid(nr, nc):
                degree = get_degree(nr, nc)
                next_positions.append((degree, nr, nc))

        # 按 degree 升序排序：优先探索"出口最少"的路径
        # 这是 Warnsdorff 规则的核心：先走最难到达的位置，
        # 避免后续陷入"死胡同"
        next_positions.sort()

        for _, nr, nc in next_positions:
            if backtrack(nr, nc, step + 1):
                return True

        # 无解，撤销选择
        board[r][c] = -1
        return False

    if backtrack(start_row, start_col, 0):
        return board
    return None


# 示例 3：骑士周游问题的纯暴力回溯版（不优化，n≥6 时极慢）
def knights_tour_bruteforce(n):
    """纯暴力回溯，仅用于对比理解 Warnsdorff 优化的效果"""
    moves = [
        (-2, -1), (-2, 1), (-1, -2), (-1, 2),
        (1, -2), (1, 2), (2, -1), (2, 1)
    ]
    board = [[-1] * n for _ in range(n)]

    def is_valid(r, c):
        return 0 <= r < n and 0 <= c < n and board[r][c] == -1

    def backtrack(r, c, step):
        board[r][c] = step
        if step == n * n - 1:
            return True

        # 暴力版：按固定顺序尝试所有 8 个方向，不排序
        for dr, dc in moves:
            nr, nc = r + dr, c + dc
            if is_valid(nr, nc):
                if backtrack(nr, nc, step + 1):
                    return True

        board[r][c] = -1
        return False

    if backtrack(0, 0, 0):
        return board
    return None
```

**骑士周游问题的关键分析**：

| 方面 | 说明 |
|------|------|
| 问题类型 | 哈密顿路径问题（HCP）的特例，NP-完全 |
| 棋盘规模 | 标准 8×8 有约 26 万亿条可能的路径，暴力不可行 |
| 朴素回溯 | O(8^(n²))，n=5 时尚可，n=6 开始极慢 |
| Warnsdorff 优化 | 在 8×8 棋盘上几乎总能在线性时间内找到解 |
| 适用算法 | 回溯 + 剪枝 / 分治 / 神经网络（非递归解法）|

**Warnsdorff 规则为什么有效**：

核心思想是**优先探索"出口最少"的位置**。直观理解：如果某位置只剩下一步可走，你必须立即走它，否则后续再也无法访问该位置。这种"先走最难走的路"的策略极大减少了回溯次数。

```
类比：走迷宫时优先走看起来分支最少的岔路，避免把死胡同留到最后
```

**时间复杂度对比**：

| n | 暴力回溯 | Warnsdorff 优化 |
|---|---------|----------------|
| 5 | 瞬间 | 瞬间 |
| 6 | 数秒~数分钟 | 瞬间 |
| 7 | 极慢 | < 0.1s |
| 8 | 不可行 | < 0.1s |

**其他优化方法**：
- **分治策略**：将棋盘划分为小块，分别求解后再拼接（如 8×8 可分解为 4×4 子问题）
- **对称性利用**：利用棋盘对称性减少搜索空间（旋转、翻转等价）
- **结合回溯与定式**：对小规模预计算，大规模用启发式
- **魔方阵预计算**：某些固定路径模式（如 8×8 的已知解）可直接查表

**回溯模板**：
```python
def backtrack(路径, 选择列表):
    if 满足结束条件:
        保存路径
        return
    for 选择 in 选择列表:
        做选择
        backtrack(路径, 选择列表)
        撤销选择
```

### 6.5 分支限界 (Branch & Bound)

回溯的优化版本，回溯使用 DFS，分支限界通常使用 BFS（优先队列），用**上界/下界函数**剪掉不可能产生更优解的分支。

**示例：0/1 背包的分支限界**：
- 按单位重量价值排序
- 每个节点计算剩余物品的最大可能价值（上界）
- 如果上界 ≤ 当前最优解，剪枝
